# Административные команды бота

## Обзор

Система банов позволяет администраторам управлять пользователями бота, блокируя нарушителей правил.

## Команды

### `/ban <user_id> <причина> [длительность]`
Банит пользователя.

**Параметры:**
- `user_id` - Telegram ID пользователя
- `причина` - Причина бана
- `длительность` - (опционально) Длительность бана в формате `Nh` (часы) или `Nd` (дни)

**Примеры:**
```
/ban 123456789 Спам
/ban 123456789 Нарушение правил 7d
/ban 123456789 Временный бан 24h
```

### `/unban <user_id>`
Разбанивает пользователя.

**Параметры:**
- `user_id` - Telegram ID пользователя

**Пример:**
```
/unban 123456789
```

### `/bans [all]`
Показывает список банов.

**Параметры:**
- `all` - (опционально) Показать все баны, включая неактивные

**Примеры:**
```
/bans
/bans all
```

### `/userinfo <user_id>`
Показывает информацию о пользователе.

**Параметры:**
- `user_id` - Telegram ID пользователя

**Пример:**
```
/userinfo 123456789
```

### `/cleanup_bans`
Очищает истекшие временные баны.

**Пример:**
```
/cleanup_bans
```

## Типы банов

### Временный бан
- Автоматически истекает через указанное время
- Пользователь может снова использовать бота после истечения
- Пример: `/ban 123456789 Спам 24h`

### Постоянный бан
- Действует до ручного разбана администратором
- Пользователь не может использовать бота
- Пример: `/ban 123456789 Нарушение правил`

## Администраторы

Текущие администраторы (Telegram ID):
- `8405113240` - Основной администратор

Для добавления новых администраторов отредактируйте список `ADMIN_IDS` в файле `handlers/admin.py`.

## Автоматическая очистка

Система автоматически очищает истекшие временные баны каждые 12 часов.

## Логирование

Все действия с банами логируются в `bot.log`:
- Создание банов
- Разбаны
- Попытки забаненных пользователей использовать бота
- Автоматическая очистка истекших банов
